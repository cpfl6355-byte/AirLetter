# Air Letter — React 변환 작업 계획서

본 계획서는 `sta/` 디렉토리의 퍼블리싱 소스(HTML, CSS, JS)를 **React (Vite) + TypeScript + styled-components** 환경으로 변환하기 위한 상세 설계 및 작업 순서입니다.

---

## 📋 1. 원본 소스 분석 요약

### 1-1. 화면 구성 (단일 SPA 구조 - 6개 Screen)
- **screen-home (홈)**: 실시간 대기질 수치(PM10/PM2.5) 원형 게이지 애니메이션, 오늘 대기질 요약 가이드 및 위험도 요약 카드.
- **screen-profile (건강 프로필 등록)**: 사용자 나이(Range), 지역(Select), 건강 상태(Multi-select Chip Group) 등록 및 저장.
- **screen-risk (위험도 분석 결과)**: 프로필과 대기질 단계를 결합한 위험도 원형 미터(1~5단계), 위험 요인 분석 및 주의사항 목록.
- **screen-guide (건강 가이드)**: 4가지 대기질 단계(좋음, 보통, 나쁨, 매우 나쁨)별 상세 행동 가이드를 탭 형식으로 조회.
- **screen-record (건강 기록)**: 일일 증상 선택(기침, 콧물 등) 및 심각도 입력(1~5단계) 후 목록에 역순으로 저장.
- **screen-settings (설정)**: 프로필 수정 바로가기, 미세먼지 알림/리포트/리마인더 토글, LocalStorage 데이터 전체 초기화.

### 1-2. 공통 UI 컴포넌트
- **Header**: 로고(✈️ Air Letter)와 우측 상단의 현재 지역 선택 버튼(홈 화면에서만 노출).
- **Bottom Navigation**: 6개 메뉴(홈, 프로필, 위험도, 가이드, 기록, 설정)의 화면 전환 네비게이션.
- **Alert Banner**: 미세먼지 단계가 '나쁨' 이상(주의보/경보 발령 시)일 때 홈 화면 상단에 알림 메시지 노출.
- **Mascot Character**: 실시간 대기질 지수에 연계하여 캐릭터의 상태(좋음, 보통, 나쁨, 매우나쁨)가 역동적으로 바뀌는 곰돌이 캐릭터 비주얼 요소.

### 1-3. 상태 관리 및 데이터 흐름
- **LocalStorage**: `profile`(사용자 나이, 지역, 건강 조건), `settings`(토글 알림 설정), `records`(증상 기록), `region`(선택된 지역)을 저장하여 새로고침 시에도 유지.
- **API 연동**: 에어코리아 미세먼지 경보 API를 호출하여 시/도 단위의 실시간 발령 정보를 파싱하고, 대기질 레벨 및 게이지를 업데이트.

## 🎨 3. 디자인 시스템 (styled-components) 이관 계획

### 3-1. 글로벌 테마 (`src/styles/theme.ts`)
기존 `style.css`의 `:root` 변수들을 리액트 테마 객체로 정의합니다.
- **Air Quality Colors**: `good` (#16A97A), `moderate` (#D4920A), `unhealthy` (#E06A10), `hazardous` (#C72828)
- **Backgrounds**: `primary` (#E8F4FD), `secondary` (#F0F8FF), `card` (rgba(255, 255, 255, 0.92))
- **Texts**: `primary` (#0D2D4A), `secondary` (#2E6088), `muted` (#6A9BB8)
- **Accent**: `accent` (#1E90D6), `glow` (rgba(30, 144, 214, 0.22))

### 3-2. 전역 스타일 (`src/styles/GlobalStyle.ts`)
- Reset CSS 적용 및 Pretendard 웹폰트 로드.
- 기존 CSS에 존재하는 키프레임 애니메이션(`fadeInUp`, `pulse`, `breathe`) 적용.
- 웹앱 전체 스케일을 최대 430px 중앙 정렬하는 모바일 쉘 레이아웃 스타일 설정.

## 📝 4. 단계별 구현 마일스톤

### [Phase 1] 프로젝트 생성 및 환경 설정
1. 기존 `sta` 폴더의 퍼블리싱 산출물을 임시 디렉토리(`sta_backup`)에 백업합니다.
2. `npx create-vite@latest ./ --template react-ts` 명령어로 Vite React-TS 환경을 설정합니다.
3. `styled-components` 및 관련 타입 패키지를 설치합니다.
4. `.env` 파일을 구성하여 API Key 및 API Base URL을 환경 변수화합니다.

### [Phase 2] 전역 스타일 및 타입 정의
1. `src/styles/theme.ts`와 `GlobalStyle.ts`를 구현하여 기존 퍼블리싱 스타일의 토대를 이관합니다.
2. `src/types/index.ts`에 사용자 프로필, 대기질 데이터, 건강 기록, 탭 네비게이션 등에 필요한 TS 인터페이스를 선언합니다.

### [Phase 3] 유틸리티 및 커스텀 훅 개발
1. `useLocalStorage` 훅을 작성하여 기존 로컬스토리지 입출력 로직(`LS.save`, `LS.load`)을 React의 상태 모델로 래핑합니다.
2. `useAirData` 훅을 개발하여 에어코리아 API로부터 비동기 데이터를 받아 로딩, 에러, 가공 데이터를 분기 처리합니다.
3. `riskCalculator.ts` 및 `airQuality.ts` 파일에 위험 지수 계산과 등급명 포맷팅 유틸 함수를 작성합니다.

### [Phase 4] 공통 컴포넌트 변환
1. **Header / Footer**: `useState`를 이용해 App 단에서 페이지 전환이 일어나도록 이벤트 프로퍼티를 구현합니다.
2. **GaugeCircle**: Canvas/SVG 및 기존 CSS 트랜지션을 사용하여 원형 게이지가 채워지는 동작과 숫자가 상승하는 카운트업 애니메이션을 리액트 라이프사이클로 구현합니다.
3. **MascotCharacter**: 실시간 대기질 등급(AirLevel)에 따라 곰돌이 이미지 분기 렌더링 및 모션 효과를 추가합니다.
   - *좋음*: 활짝 웃는 곰돌이 (`bear_good.png`)
   - *보통*: 은은한 미소의 곰돌이 (`bear_moderate.png`)
   - *나쁨*: 마스크를 쓴 곰돌이 (`bear_unhealthy.png`)
   - *매우 나쁨*: 울먹이며 무거운 KF99 마스크를 쓴 곰돌이 (`bear_hazardous.png`)
4. **기타 UI 컴포넌트**: `LevelBadge`, `ChipGroup`, `ToggleSwitch` 등을 styled-components의 상태 기반 분기를 활용하여 재생산합니다.

### [Phase 5] 메인 페이지 구현 및 연동
1. **HomePage**: 대기질 데이터와 연동하여 실시간 대기 상태를 실시간 게이지로 렌더링하고, **대기질 게이지 영역 사이 또는 하단에 `MascotCharacter` 컴포넌트를 배치**하여 직관적으로 날씨와 위험도를 체감할 수 있도록 구성합니다. 등록된 프로필 유무에 따라 조건부 렌더링을 처리합니다.
2. **ProfilePage**: Controlled Component 형식으로 나이, 지역 및 체크박스 상태를 리액트 state로 결합하고 프로필 데이터를 저장 및 갱신합니다.
3. **RiskPage / GuidePage / RecordPage / SettingsPage**: 각 페이지의 화면 레이아웃과 이벤트를 이식하고 비즈니스 로직(증상 입력 추가, 로컬스토리지 초기화 등)을 통합합니다.

### [Phase 6] 최적화 및 최종 검증
1. styled-components 테마 변수가 누락 없이 정상 적용되었는지 확인합니다.
2. 화면 간 슬라이드 애니메이션 및 탭 포커스가 의도대로 잘 나타나는지 브라우저에서 최종 테스트를 진행합니다.
